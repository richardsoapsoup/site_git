O que é o TDD?

"TDD significa Desenvolvimento Orientado por Testes (Test Driven Development), e trata-se de uma prática de desenvolvimento de software onde a codificação das funcionalidades começa a partir da escrita de testes unitários."

Tipos de teste

Teste de unidade:

  • É focado na menor unidade de projeto: componente ou módulo.

  • O módulo é testado para garantir que as informações fluem corretamente para dentro e para fora do módulo.

  • São considerados uma etapa auxiliar da codificação.



Teste de integração:

  • Se os módulos funcionam independentemente, por que duvidar se eles funcionam juntos?

  • A ideia do teste de integração é testar a cada “pedaço” de software construído.

  • Evitar o big bang: juntar tudo e testar tudo junto.



Teste alfa e beta:

  • É praticamente impossível para o desenvolvedor prever como o cliente vai utilizar o sistema.

  • Os testes de aceitação validam os requisitos. Porém, às vezes esses testes podem demorar muito, e desencadear em outros erros.

  • Por esse motivo, são usados testes alfa e beta para descobrir problemas que somente o usuário seria capaz de encontrar.



O que é o pytest?

O pytest é uma framework de teste para python que provê soluções para executar testes e fazer validações diversas, com a possibilidade de estender com plugins e executar testes do unittest no python.
