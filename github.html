O que é o controle de versões?

"Controle de versão é um sistema que registra as mudanças feitas em um arquivo ou conjunto de arquivos ao longo do tempo de forma que você possa recuperar versões específicas"

O controle de versão não é necessariamente utilizado apenas na programação, podendo também ser usado na edição de documentos de texto, imagens, vídeos e projetos diversos.

Tipos de sistema de controle

Sistema local:

  • Os sistemas de controle de versão locais, armazenam em uma máquina todas as alterações em arquivos sob controle de revisão.

  • O rcs armazena as diferenças entre cada versão de um arquivo, e permite que sejam retornados à versões anteriores.

  • Exemplo: rcs (Revision Control System)



Sistema centralizado:

  • Permite que diversos desenvolvedores possam trabalhar juntos em um mesmo projeto.

  • Possui um servidor central que armazena as versões dos arquivos, os clientes fazem cópias desses arquivos, sua principal desvantagem é a dependência do servidor

  • Exemplos: CVS, Subversion



Sistema distribuido:

  • Ao invés de fazer cópias das últimas versões, todo o repositório é copiado. Assim, qualquer máquina pode ajudar o servidor a se recuperar.

  • Permite o trabalho em conjunto com diversas pessoas e a criação de fluxos de trabalhos diferentes do que são usados nos sistemas centralizados.

  • Exemplos: git, mercural, Darcs.



O que é o git?

O git é um sistema de controle de versões distribuído, usado principalmente no desenvolvimento de software, mas pode ser usado para registrar o histórico de edições de qualquer tipo de arquivo.

Como funciona o git?

Uma das principais características do git está no fato de armazenar diferenças entre os arquivos como a maioria dos VCS faz, são armazenados snapshots que funcionam como uma fotografia daquele sistema de arquivos naquele momento.

Todos os commits possuem checksum, isso garante integridade já que antes de armazenar as informações essa soma é calculada e podem ser identificados problemas.

Os arquivos do git estão sempre em um dos três estados:

Consolidado (commited) - Estão armazenados em sua base local

Modificado (modified) - Arquivo sofreu mudança mas não foi commitado

Preparado (staged) - Arquivo marcado para fazer parte do próximo snapshot

Principais comandos do git:

  • Git init(inicialização)

  • Git status(checar status)

  • Git add(realizar alteração)

  • Git commit(finalizar ação)



O que é o github?

O GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git. Ele permite que programadores, utilitários ou qualquer usuário cadastrado na plataforma contribuam em projetos privados e/ou Open Source de qualquer lugar do mundo.
